title=Detailed changelog for Adblock Plus 1.3

{{s1 The following lists the major changes compared to Adblock Plus 1.2.2.}}

{{s2 User interface}}

  • {{s3 New translation: Mexican Spanish}}
  • {{s4 Removed unmaintained translations: Indonesian, Malay, Brasilian Portuguese}}
  • {{s5 Added an issue reporter allowing to send reports about Adblock Plus issues on web pages ("Report issue on this page" in ABP icon's menu, forum topic)}}
  • {{s6 Image maps can now be blocked from context menu like regular images (bug 16605)}}
  • {{s7 Links to documentation on adblockplus.org will automatically open in user's language if available}}

{{s8 ABP icon}}

  • {{s9 Clicking the icon will now open menu by default (old behavior can be restored by changing extensions.adblockplus.defaulttoolbaraction preference)}}
  • {{s10 First item in menu is the new issue reporter (forum topic)}}
  • {{s11 Disabled filters and filter subscriptions are no longer counted in the tooltip}}

{{s12 List of blockable items}}

  • {{s13 Selecting "Open in New Tab" from context menu will open multiple tabs if multiple items are selected (forum topic)}}
  • {{s14 Document source column shows whether the request was made to a first-party or third-party location, filtering and sorting by this information is possible (forum topic)}}
  • {{s15 Tooltip now shows info on disabled filters as well (forum topic)}}
  • {{s16 Fixed: Scrolling to selected item broken in Firefox 4}}
  • {{s17 Fixed "timed textboxes are deprecated" warning}}

{{s18 Filter composer}}

  • {{s19 Suggestions no longer distinguish between protocols (forum topic)}}
  • {{s20 A warning is displayed if a custom pattern no longer matches the address it is supposed to block (forum topic)}}

{{s21 "Add filter subscription" dialog}}

  • {{s22 Removed warning displayed when Adblock or Filterset.G are installed (both have a negligible install base by now)}}
  • {{s23 Made sure that the buttons aren't too far below subscription selection when opened in browser after first installation}}

{{s24 Preferences window}}

  • {{s25 A snail icon is displayed for slow filters (was previously an exclamation mark, forum topic)}}
  • {{s26 Triggering a filter subscription update manually always downloads the file, even if server claims that it didn't change}}
  • {{s27 No longer storing user's choice to collapse a subscription for subscriptions that have been removed}}

{{s28 Core}}

  • {{s29 Dropped support for Firefox 3.0, Songbird 1.8 and Midbrowser}}
  • {{s30 Improved implementation, made it cleaner and more modular (also changed the way how other extensions should access Adblock Plus)}}
  • {{s31 Removed separation between background images and "regular" images, $background and $image options are identical now (reliably distinguishing the two is no longer possible)}}
  • {{s32 Made sure that filters added by other extensions like Element Hiding Helper are always enabled (forum topic)}}
  • {{s33 Added file default/patterns.ini to the extension, can be changed to define default filters or filter subscriptions for new users}} {{s34 (forum topic)}}
  • {{s35 Filter subscriptions added by other extensions will no longer persist, need to be added again on each browser start (ensures that these subscriptions don't stay around if the corresponding extension is uninstalled)}}
  • {{s36 Removed code importing preferences and filters from Adblock and Adblock Plus 0.5 (both have a negligible install base by now)}}

{{s37 Performance}}

  • {{s38 Changed the way Adblock Plus stores its data to improve performance and memory use}}
  • {{s39 Improved filter matching performance by 20-35%}}
  • {{s40 Reduced time required to set up element hiding on browser startup}}
  • {{s41 Using a more efficient approach to preference management now}}

{{s42 Subscription downloads}}

  • {{s43 Using better algorithm to determine when to download subscriptions (blog post)}}
  • {{s44 Added precautions for the case that computer's clock is skewed}}
  • {{s45 Last modification time is reset now when a subscription redirects to a new location}}

{{s46 Changelog for the previous release}}
