title=Detailed changelog for Adblock Plus 1.2

{{s1 The following lists the major changes compared to Adblock Plus 1.1.3. If you experience issues with this release please check the list of known issues.}}

{{s2 User interface}}

  • {{s3 New translations: Bulgarian, Argentinian Spanish}}
  • {{s4 Removed unmaintained translations: Estonian, Persian, Norwegian (Bokm√•l), Lithuanian}}
  • {{s5 Blocking HTML5 audio/video is now possible via context menu}}
  • {{s6 Added more information to the ABP icon tooltip (forum topic)}}
  • {{s7 Fixed: Some Adblock Plus messages were appearing with the generic "JavaScript Application" title (forum topic)}}
  • {{s8 Simplified status bar panel by removing the hidden "Adblock" text (user style updated for those who still want to see it)}}

{{s9 "Add filter subscription" dialog (also displayed as post-install page)}}

  • {{s10 Completely redesigned to make this dialog easier to use (forum topic)}}
  • {{s11 Best available subscription recommendation is preselected}}
  • {{s12 In the post-install case: added warning when leaving without adding a filter subscription}}
  • {{s13 Adding "custom" subscriptions is possible in the same dialog now}}
  • {{s14 The complete list of subscriptions loads into the same dialog, visiting the webpage is no longer necessary}}
  • {{s15 Filter subscriptions matching user's locale are highlighted}}
  • {{s16 For supplemental subscriptions: Adding main subscription is suggested if the user isn't subscribed to it yet. Specifying main subscription is also possible with abp: links in webpages (forum topic)}}
  • {{s17 Fixed: "Add filter subscription" page on first install might be closed by session restore}}

{{s18 "Tabs on Flash and Java" feature}}

  • {{s19 Improved positioning of tabs, position now adjusts to dynamic changes in the webpage and scrolling}}
  • {{s20 Fixed most conditions where tabs would become invisible due to page layout (bug 12910)}}
  • {{s21 Fixed cases where tabs would cause side-effects when showing up (e.g. bug 22246)}}
  • {{s22 Fixed cases where tabs break assumptions made by webpages, particularly Acid3 test (bug 22435)}}
  • {{s23 Tabs now fade out when mouse is moved away instead of disappearing immediately}}

{{s24 List of blockable items}}

  • {{s25 When new tabs are opened browser preferences are respected (particularly whether the tab should be opened in background)}}
  • {{s26 Fixed: Context menu keyboard key wasn't working on the list}}
  • {{s27 Fixed: Search string wasn't remembered when detaching/reattaching (forum topic)}}
  • {{s28 "Disable filter on ..." was mistakenly hidden for some filters (bug 22481)}}
  • {{s29 No longer listing disabled filters if Adblock Plus is disabled (forum topic)}}
  • {{s30 Ensured correct functioning even with browser.frames.enabled preference switched off}}

{{s31 Preferences window}}

  • {{s32 Filter groups like "My Ad Blocking Rules" are automatically re-enabled when the last filter is removed from them (forum topic)}}
  • {{s33 Fixed: Preferences window broken on operating systems with "exotic" regional settings like Georgian (work-around for Mozilla bug 508783)}}
  • {{s34 Fixed: Context menu keyboard key wasn't working on the list}}
  • {{s35 Fixed: Hotkeys like Ctrl+X sometimes wouldn't work until the context menu is opened}}

{{s36 Filter composer}}

  • {{s37 Added "block entire domain" as filter suggestion in addition to "block this server" (forum topic)}}

{{s38 "About Adblock Plus" dialog}}

  • {{s39 Changed presentation of the contributors list to make it more compact, significantly less scrolling necessary now}}
  • {{s40 Most parts of the contributors list are populated dynamically now (filter subscription authors from the list of recommended subscriptions, translators from the info in extension's install manifest)}}

{{s41 Application support}}

  • {{s42 Firefox Mobile/Fennec: Adblock Plus status is displayed in identity panel or as a site action (Fennec 1.0 and Fennec 1.1 respectively) and allows enabling/disabling Adblock Plus on a site}}
  • {{s43 Firefox Mobile/Fennec: Made post-install page show up reliably (Mozilla bug 531071)}}
  • {{s44 Thunderbird: Positioned toolbar icon with the message actions, that's the more logical place for it}}
  • {{s45 Thunderbird: Made tabs opened from Adblock Plus dialogs open in Thunderbird rather than external browser}}
  • {{s46 SeaMonkey Mail: Opening tabs while no browser window is open now opens a new browser window (same way links in mail are treated)}}
  • {{s47 Songbird: Fixed toolbar icon in the new Purplerain theme (remaining issue is Songbird bug 20787)}}

{{s48 Core}}

  • {{s49 Implemented a number of performance improvement measures, especially to reduce application startup delay}}
  • {{s50 Added new $elemhide filter option for exception rules, allows disabling all element hiding rules on a page (forum topic)}}
  • {{s51 Subscription download errors are reported to Error Console}}
  • {{s52 Fixed: loading filter rules with separator placeholder (^) is very slow}}
