title=Detailed changelog for Adblock Plus 1.1.2
  • {{s1 Minimal Fennec 1.0 support (at least Beta 6 required)}}
  • {{s2 Made subscription downloads support load balancing, downloads can be distributed to several servers now}}
  • {{s3 Requests for fallback URL after multiple failed subscription downloads now contain information about the reason of the failure}}
  • {{s4 Subscription downloads no longer fail if network.http.redirection-limit preference is misconfigured}}
  • {{s5 List of filter subscriptions no longer offered only on first installation, also offered on update if the user has neither filter subscriptions nor own filters}}
  • {{s6 Improved recognition of document source for $domain option in the filters (bug 22007)}}
  • {{s7 Fixed: Menu View / Adblock Plus: Blockable items isn't always correctly reflecting whether list of blockable items is open (bug 21994)}}
  • {{s8 Fixed: Position of detached list of blockable items not restored (Firefox 3.7 nightlies)}}
  • {{s9 Fixed: Blocked object stays blocked even if opened in a new tab}}
  • {{s10 Updated compatibility range: Firefox 3.0.12 to 3.6, SeaMonkey 2.0, Thunderbird 3.0, Songbird 1.2.0 to 1.4.x.}}

{{s11 Known issues}}

  • {{s12 The extension Ask Toolbar 2.1.0.5 (usually installed under names Zone Alarm Toolbar, Foxit Toolbar, Vuze Toolbar) causes Adblock Plus and other extensions/the browser to malfunction (bug 530747). Solution: uninstall Ask Toolbar.}}
  • {{s13 Skype Extension 2.2.0.102 causes Adblock Plus to malfunction, browser functionality is affected as well. Solution: uninstall Skype Extension.}}
  • {{s14 Ecosia extension in version 1.0.3 removes all Adblock Plus filter subscriptions (bug 533419). Solution: update to Ecosia 1.0.4 and add a filter subscription again.}}
  • {{s16 Firefox/Thunderbird/SeaMonkey builds from some Linux distributions (not the official builds however) will refuse to install Adblock Plus with the message: "Signing could not be verified". Solution: update "nss" package to the latest version available for your distribution. If that doesn't help, view certificates in the encryption/certificates tab of the options, select "StartCom Certification Authority" under "Authorities" and click "Edit". Make sure "This certificate can identify software makers" is checked and click OK.}}
