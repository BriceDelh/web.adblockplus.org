title=Detailed changelog for Adblock Plus 0.7.2.3
  • {{s1 Added Persian translation}}
  • {{s2 SeaMonkey: Major security issue fixed — switched on XPCNativeWrappers that aren't used by default for some reason}}
  • {{s3 Improved storage of internal data, old way was less efficient and potentially insecure (bug 15758)}}
  • {{s4 Made sure that web pages constantly reloading images won't increase memory usage (bug 15317)}}
  • {{s5 "Show tabs on Flash and Java" again can be used when JavaScript is disabled (bug 15637)}}
  • {{s6 Made it harder for web sites to tamper with tabs on objects like Flash or Java}}
  • {{s7 Made tab displayed by "Show tabs on Flash and Java" a link (because of the context menu and ability to middle-click, bug 15638)}}
  • {{s8 Improved check for objects opened on its own (tabs shouldn't be shown for these objects)}}
  • {{s9 Toolbar icon now looses its drop marker in vertical toolbars like the one created by All-In-One Sidebar extension to prevent resizing of the toolbar, menu available with a right-click then (forum thread)}}
  • {{s10 Firefox: Default placement of the toolbar icon is now the right edge of the navigation toolbar (forum thread)}}
  • {{s11 "Disable Adblock Plus on ..." options are no longer disabled when Preferences dialog is open}}
  • {{s12 Added methods addPatterns() and removePatterns() to the public IAdblockPlus interface (can be used from other extensions)}}
  • {{s13 Using Gecko's built-in feature for scrolling to node when selecting in the sidebar — should be less error-prone}}
  • {{s14 No longer relying on extension version reported by the browser (is unreliable in SeaMonkey, forum thread)}}
  • {{s15 Can be installed in Songbird 0.3 now}}
  • {{s16 Removed obsolete file property from install manifest (not required for Firefox 1.5 and higher)}}
