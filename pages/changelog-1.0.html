title=Detailed changelog for Adblock Plus 1.0

{{s1 The following lists the major changes compared to Adblock Plus 0.7.5.4:}}

{{s2 Known issues}}

  • {{s3 Format of the filter storage changed so going back to a previous version requires exporting the filters and re-importing them after the downgrade}}
  • {{s4 Mac OS X: Flash movies might not be displayed properly if "Show tabs on Flash and Java" option is switched on}}
  • {{s5 Some translations contain untranslated English strings}}

{{s6 New features}}

  • {{s7 New filter composer dialog makes adding filters easier}}
  • {{s8 New filter options}} {{s9 $third-party}} {{s10 and}} {{s11 $~third-party}} {{s12 restrict filters to third-party/first-party requests (forum topic)}}

{{s13 User interface}}

  • {{s14 Added translations: Armenian, Galician, Indonesian}}

{{s15 List of blockable items}}

  • {{s16 Tooltip now shows the filter source in addition to the filter itself (forum topic)}}
  • {{s17 Filters can now be disabled and re-enabled from the context menu (forum topic)}}
  • {{s18 Changed default action for blocked items to "Add exception rule" instead of "Edit filter"}}
  • {{s19 Made "State" the default sort order (blocked items first)}}
  • {{s20 Added support for multiple selection and copying of multiple addresses/filters (forum topic)}}
  • {{s21 Added "Select all" to the context menu}}
  • {{s22 Added "Copy filter" item to the context menu (forum topic)}}
  • {{s23 Added "Size" column (hidden by default), same information shown in the tooltip}}
  • {{s24 Changed filter color for element hiding hits (forum topic)}}
  • {{s25 Fixed: Header is hidden in full screen mode}}

{{s26 Preferences window}}

  • {{s27 Eliminated delay when window is opened and much of the delay when changes are applied}}
  • {{s28 Mac OS X: Preferences menu changed into toolbar to make sure it stays inside the window (forum topic)}}
  • {{s29 Changed names for groups of user-defined filters (forum topic)}}
  • {{s30 Group/subscription titles use bold text now (forum topic)}}
  • {{s31 Added "View" menu to control visible columns and sorting, removed column picker}}
  • {{s32 Made new "View" menu the context menu for list header}}
  • {{s33 Groups containing user-defined filters can now be disabled (forum topic)}}
  • {{s34 Images for "Enabled" column made more obvious (forum topic, forum topic)}}
  • {{s35 Removed "Add filter" dummy from the list (forum topic)}}
  • {{s36 Added support for comments in all filter groups, not only "Advertisement filters"}}
  • {{s37 Comments now always stay before the same filter, regardless of sorting (also, they can be dragged even if the list is sorted)}}
  • {{s38 For invalid filters, tooltip now explains why they are invalid}}
  • {{s39 Removed user interface elements that were made obsolete by the filter composer: regular expression warning and corresponding preference}} {{s40 extensions.adblockplus.warnregexp}}{{s41 , dropdown list in filter editor (forum topic)}}
  • {{s42 Filter export now inserts a checksum comment into the file (forum topic)}}
  • {{s43 Added a tooltip explaining the meaning of the exclamation mark}}
  • {{s44 Added a way to disable/enable filters and subscriptions from context menu}}
  • {{s45 "Reset filter statistics" now resets "Last hit" as well (bug 18650)}}
  • {{s46 Fixed: Findbar in preferences dialog has no icons (Firefox 3)}}
  • {{s47 Fixed: Dates would sometimes use wrong character encoding (bug 441370)}}
  • {{s48 Fixed: "Edit filter in effect" might select a filter in a disabled subscription even though it exists in enabled subscriptions as well (forum topic)}}
  • {{s49 Fixed: Clicking twice on the Enabled column starts filter editor}}
  • {{s50 Fixed: Changes to "download automatically" status are not displayed immediately}}
  • {{s51 Pasting filters with spaces (in particular comments) now possible (bug 18920)}}

{{s52 Filter subscription selection}}

  • {{s53 First start: Filter subscription selection is opened in a new tab rather than in a separate window (bug 15786)}}
  • {{s54 First start: Filter subscription selection will show up even if other extensions already added a filter subscription they maintain.}}
  • {{s55 "Add filter subscription" from Preferences brings up filter subscription selection rather than the dialog for custom filter subscriptions.}}
  • {{s56 Added warning if Filterset.G is installed with option to uninstall automatically}}
  • {{s57 Added Adblock warning with option to uninstall automatically (this was a separate question pop-up before)}}

{{s58 Other user interface changes}}

  • {{s59 Regular expression warning was extended to also warn for filters that are too short to be optimized}}
  • {{s60 Warning that Adblock Plus is disabled allows re-enabling with a single click}}
  • {{s61 Removed ancient Drag&Drop handling code (not discoverable and not advertised anywhere)}}
  • {{s62 Fixed: "Disable on this page only" won’t work for pages with query string}}
  • {{s63 About Adblock Plus: Made version number selectable so that it can be copied}}
  • {{s64 About Adblock Plus: Made homepage link localizable (and different for German/French locales, bug 17136)}}
  • {{s65 Added "Adblock Plus: Blockable Items" to the View menu in Firefox (View – Toolbars menu in SeaMonkey)}}

{{s66 Core}}

  • {{s67 Core code was redesigned for better maintainability, code documentation and automated tests (based on}} {{s69 Mochitest}}{{s70 ) were added for large parts of the core}}
  • {{s71 Made sure filter storage file isn't overwritten unless write was successful (bug 16488)}}
  • {{s72 Improved performance (esp. reading filter from disk and saving filters for large filter lists), reduced the amount of memory used}}
  • {{s73 Fixed: Exception rules don’t always apply to element hiding in Firefox 3 (bug 20034)}}
  • {{s74 Firefox 1.5, Thunderbird 1.5, SeaMonkey 1.0, Songbird 0.5/0.6 are no longer supported (removed some backwards compatibility hacks)}}
  • {{s75 Flock 1.2/2.0 and Songbird 0.7/1.0 are supported now}}
  • {{s76 Added support for Prism 0.9 (forum topic)}}
  • {{s77 Added hidden preference extensions.adblockplus.savestats to allow turning off hit statistics (useful if patterns.ini is put into a version control system) (bug 18650)}}
  • {{s78 Added support for private browsing mode in Firefox 3.1, hit statistics are disabled automatically}}
  • {{s79 All first run actions are now controlled by extensions.adblockplus.currentversion preference (replaces}} {{s80 showsubscriptions}}{{s81 ,}} {{s82 checkedtoolbar}}{{s83 ,}} {{s84 checkedadblockinstalled}} {{s85 preferences)}}
  • {{s86 Changed patterns.ini format to allow user-defined comments (and potentially other filters as well) to be stored in any filter group}}
  • {{s87 Adblock Plus 0.6 preferences are no longer imported on upgrade}}
  • {{s88 Adblock preferences are no longer imported on upgrade (filter list still imported)}}
  • {{s89 Fixed: Object tabs show up on print (bug 19405)}}
  • {{s90 Fixed: "getBoxObjectFor() is deprecated" warning in error console (bug 19126)}}

{{s91 Subscription downloads}}

  • {{s92 For subscription downloads, a comment "! Checksum: abcd" is now interpreted and files with wrong checksum are rejected to prevent broken downloads (forum topic, reference implementation to add checksums)}}
  • {{s93 Next attempt after failed download is after one day now rather than after one hour to prevent server overload (adjusted preference extensions.adblockplus.subscriptions_fallbackerrors accordingly)}}
  • {{s94 Fallback request can now disable automatic downloads for subscriptions that are no longer available}}
  • {{s95 Fixed: Subscription downloads sometimes go to browser cache}}
