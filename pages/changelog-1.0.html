title=Detailed changelog for Adblock Plus 1.0

{{s1 The following lists the major changes compared to Adblock Plus 0.7.5.4:}}

{{s2 Known issues}}

  • {{s3 Format of the filter storage changed so going back to a previous version requires exporting the filters and re-importing them after the downgrade}}
  • {{s4 Mac OS X: Flash movies might not be displayed properly if "Show tabs on Flash and Java" option is switched on}}
  • {{s5 Some translations contain untranslated English strings}}

{{s6 New features}}

  • {{s7 New filter composer dialog makes adding filters easier}}
  • {{s8 New filter options $third-party and $~third-party restrict filters to third-party/first-party requests (forum topic)}}

{{s9 User interface}}

  • {{s10 Added translations: Armenian, Galician, Indonesian}}

{{s11 List of blockable items}}

  • {{s12 Tooltip now shows the filter source in addition to the filter itself (forum topic)}}
  • {{s13 Filters can now be disabled and re-enabled from the context menu (forum topic)}}
  • {{s14 Changed default action for blocked items to "Add exception rule" instead of "Edit filter"}}
  • {{s15 Made "State" the default sort order (blocked items first)}}
  • {{s16 Added support for multiple selection and copying of multiple addresses/filters (forum topic)}}
  • {{s17 Added "Select all" to the context menu}}
  • {{s18 Added "Copy filter" item to the context menu (forum topic)}}
  • {{s19 Added "Size" column (hidden by default), same information shown in the tooltip}}
  • {{s20 Changed filter color for element hiding hits (forum topic)}}
  • {{s21 Fixed: Header is hidden in full screen mode}}

{{s22 Preferences window}}

  • {{s23 Eliminated delay when window is opened and much of the delay when changes are applied}}
  • {{s24 Mac OS X: Preferences menu changed into toolbar to make sure it stays inside the window (forum topic)}}
  • {{s25 Changed names for groups of user-defined filters (forum topic)}}
  • {{s26 Group/subscription titles use bold text now (forum topic)}}
  • {{s27 Added "View" menu to control visible columns and sorting, removed column picker}}
  • {{s28 Made new "View" menu the context menu for list header}}
  • {{s29 Groups containing user-defined filters can now be disabled (forum topic)}}
  • {{s30 Images for "Enabled" column made more obvious (forum topic, forum topic)}}
  • {{s31 Removed "Add filter" dummy from the list (forum topic)}}
  • {{s32 Added support for comments in all filter groups, not only "Advertisement filters"}}
  • {{s33 Comments now always stay before the same filter, regardless of sorting (also, they can be dragged even if the list is sorted)}}
  • {{s34 For invalid filters, tooltip now explains why they are invalid}}
  • {{s35 Removed user interface elements that were made obsolete by the filter composer: regular expression warning and corresponding preference extensions.adblockplus.warnregexp, dropdown list in filter editor (forum topic)}}
  • {{s36 Filter export now inserts a checksum comment into the file (forum topic)}}
  • {{s37 Added a tooltip explaining the meaning of the exclamation mark}}
  • {{s38 Added a way to disable/enable filters and subscriptions from context menu}}
  • {{s39 "Reset filter statistics" now resets "Last hit" as well (bug 18650)}}
  • {{s40 Fixed: Findbar in preferences dialog has no icons (Firefox 3)}}
  • {{s41 Fixed: Dates would sometimes use wrong character encoding (bug 441370)}}
  • {{s42 Fixed: "Edit filter in effect" might select a filter in a disabled subscription even though it exists in enabled subscriptions as well (forum topic)}}
  • {{s43 Fixed: Clicking twice on the Enabled column starts filter editor}}
  • {{s44 Fixed: Changes to "download automatically" status are not displayed immediately}}
  • {{s45 Pasting filters with spaces (in particular comments) now possible (bug 18920)}}

{{s46 Filter subscription selection}}

  • {{s47 First start: Filter subscription selection is opened in a new tab rather than in a separate window (bug 15786)}}
  • {{s48 First start: Filter subscription selection will show up even if other extensions already added a filter subscription they maintain.}}
  • {{s49 "Add filter subscription" from Preferences brings up filter subscription selection rather than the dialog for custom filter subscriptions.}}
  • {{s50 Added warning if Filterset.G is installed with option to uninstall automatically}}
  • {{s51 Added Adblock warning with option to uninstall automatically (this was a separate question pop-up before)}}

{{s52 Other user interface changes}}

  • {{s53 Regular expression warning was extended to also warn for filters that are too short to be optimized}}
  • {{s54 Warning that Adblock Plus is disabled allows re-enabling with a single click}}
  • {{s55 Removed ancient Drag&Drop handling code (not discoverable and not advertised anywhere)}}
  • {{s56 Fixed: "Disable on this page only" won’t work for pages with query string}}
  • {{s57 About Adblock Plus: Made version number selectable so that it can be copied}}
  • {{s58 About Adblock Plus: Made homepage link localizable (and different for German/French locales, bug 17136)}}
  • {{s59 Added "Adblock Plus: Blockable Items" to the View menu in Firefox (View – Toolbars menu in SeaMonkey)}}

{{s60 Core}}

  • {{s62 Core code was redesigned for better maintainability, code documentation and automated tests (based on Mochitest) were added for large parts of the core}}
  • {{s63 Made sure filter storage file isn't overwritten unless write was successful (bug 16488)}}
  • {{s64 Improved performance (esp. reading filter from disk and saving filters for large filter lists), reduced the amount of memory used}}
  • {{s65 Fixed: Exception rules don’t always apply to element hiding in Firefox 3 (bug 20034)}}
  • {{s66 Firefox 1.5, Thunderbird 1.5, SeaMonkey 1.0, Songbird 0.5/0.6 are no longer supported (removed some backwards compatibility hacks)}}
  • {{s67 Flock 1.2/2.0 and Songbird 0.7/1.0 are supported now}}
  • {{s68 Added support for Prism 0.9 (forum topic)}}
  • {{s69 Added hidden preference extensions.adblockplus.savestats to allow turning off hit statistics (useful if patterns.ini is put into a version control system) (bug 18650)}}
  • {{s70 Added support for private browsing mode in Firefox 3.1, hit statistics are disabled automatically}}
  • {{s71 All first run actions are now controlled by extensions.adblockplus.currentversion preference (replaces showsubscriptions, checkedtoolbar, checkedadblockinstalled preferences)}}
  • {{s72 Changed patterns.ini format to allow user-defined comments (and potentially other filters as well) to be stored in any filter group}}
  • {{s73 Adblock Plus 0.6 preferences are no longer imported on upgrade}}
  • {{s74 Adblock preferences are no longer imported on upgrade (filter list still imported)}}
  • {{s75 Fixed: Object tabs show up on print (bug 19405)}}
  • {{s76 Fixed: "getBoxObjectFor() is deprecated" warning in error console (bug 19126)}}

{{s77 Subscription downloads}}

  • {{s78 For subscription downloads, a comment "! Checksum: abcd" is now interpreted and files with wrong checksum are rejected to prevent broken downloads (forum topic, reference implementation to add checksums)}}
  • {{s79 Next attempt after failed download is after one day now rather than after one hour to prevent server overload (adjusted preference extensions.adblockplus.subscriptions_fallbackerrors accordingly)}}
  • {{s80 Fallback request can now disable automatic downloads for subscriptions that are no longer available}}
  • {{s81 Fixed: Subscription downloads sometimes go to browser cache}}
