title=Detailed changelog for Adblock Plus 1.3.5

{{s1 The following lists the major changes compared to Adblock Plus 1.3.3.}} {{s2 If you experience issues with this release please check the list of known issues.}}

{{s3 Features}}

  • {{s4 Firefox Mobile 4 is now supported, support for Firefox Mobile 1.0 has been dropped.}}
  • {{s5 K-Meleon 1.6 is now mostly supported (special builds required).}}
  • {{s6 Added support for Do-Not-Track header proposal (DNT), a filter like}} *$donottrack {{s7 will cause the header to be sent to all locations (}}{{s8 forum topic}}{{s9 ).}}
  • {{s10 Added an experimental "Recommend us on Facebook" button to the menu allowing easily recommending Adblock Plus — only displayed to Facebook users.}}

{{s11 User interface}}

  • {{s12 Malay and Estonian translations are included again.}}
  • {{s13 Allowed shortcut key preferences to have multiple values, the first available shortcut key is used.}} {{s14 Changed default Preferences key to Ctrl-Shift-E (used in Firefox 3.x), Ctrl-Shift-F (used in Firefox 4) or Ctrl-Shift-O (Firefox 4 on Mac OS X).}}
  • {{s15 Special icon state for "whitelisted" websites has been removed, the icon is now showing as "disabled" on such sites as well (forum topic).}}
  • {{s16 "Enable Adblock Plus" menu item has been renamed into "Disable everywhere".}}
  • {{s17 Firefox Mobile: First-run page is now more suitable for a small screen.}}
  • {{s18 Fixed: Contents of first-run page are sometimes cut off (forum topic).}}
  • {{s19 Fixed: "Edit filter in effect" in blockable items doesn't work correctly if Preferences dialog is already open.}}

{{s20 Issue reporter}}

  • {{s21 Disallowed reports with too many filter subscriptions.}}
  • {{s22 Disallowed reports for certain types of pages (in particular blank tabs, e-mails, local pages, extension pages).}}
  • {{s23 Thunderbird: Better reporting for blog posts, blog post address is now being sent instead of a meaningless mailbox address.}}
  • {{s24 Fixed: Disabled filter subscriptions should not be included in the report (forum topic).}}

{{s25 Core}}

  • {{s26 Implemented a faster filter matching algorithm (forum topic).}}
  • {{s27 patterns.ini file will only be saved on shutdown if some data actually changed.}}
  • {{s28 Matching and element hiding data are now cached in the Adblock Plus data directory to reduce startup time.}}
  • {{s29 Changed the way patterns.ini file is written to prevent data loss even if computer is switched off abruptly.}}
  • {{s30 Corrupted patterns.ini files are now recognized, a backup file is loaded automatically if available.}}
  • {{s31 Fixed: Element hiding rules might occasionally cause crashes (regression in Adblock Plus 1.3.3).}}
  • {{s32 Fixed: Disabling filters from the list of blockable items doesn't actually disable the filter in Firefox 3.6.}}
